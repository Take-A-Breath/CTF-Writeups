# Bandit Level 9 → Level 10
## Level Goal
The password for the next level is stored in the file data.txt in one of the few human-readable strings, preceded by several ‘=’ characters.

### Commands you may need to solve this level
```grep, sort, uniq, strings, base64, tr, tar, gzip, bzip2, xxd```

#### Method
1. command: `file data.txt`
  1. output: `data.txt: data`
2. strings data.txt
  2. All strings listed in the file (won't bother you with the out  put)
3. egrep "=+" - See table below (from [Regular Expressions - The Last Guide](https://medium.com/tech-tajawal/regular-expressions-the-last-guide-6800283ac034) 

| Meta Character | Description                                                                                           |
|----------------|-------------------------------------------------------------------------------------------------------|
| .              | Period matches any single character except a line break.                                              |
| []             | Character class. Matches any character contained between the square brackets.                         |
| [^]            | Negated character class. Matches any character that is not contained between the square brackets      |
| *              | Matches 0 or more repetitions of the preceding symbol.                                                |
| +              | Matces 1 or more repetitions of the preceding symbol.                                                 |
| ?              | Makes the preceding symbol optional.                                                                  |
| {n,m}          | Character group. Matches the characters xyz in that exact order.                                      |
| (xyz)          | Alternation. Matches either the characters before or the characters after the symbol.                 |
| \|             | Alternation. Matches either the characters before or the characters after the symbol.                 |
| \              | Escapes the next character. This allows you to match reserved characters [ ] ( ) { } . * + ? ^ $ \ \| |
| ^              | Matches the beginning of the input.                                                                   |
| $              | Matches the end of the input.                                                                         |

Answer: truKLdjsbJ5g7yyJ2X2R0o3a5HQJFuLk

*Note: you can winnow down the output even further by adding = to the `egrep` REGEX, i.e., `egrep "==+"*
